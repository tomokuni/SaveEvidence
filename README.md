## 1. 概要

SaveEvidence は Windows デスクトップ上の画面をキャプチャし、プレビュー・編集・保存するためのアプリケーションです。  
スクリーン全体、特定ウィンドウ、任意の矩形領域の 3 種類のキャプチャ方法をサポートし、グローバルホットキーから瞬時に起動できます。

キャプチャした画像はアプリ内でプレビュー表示され、切り出し（クロップ）、クリップボード連携、ファイル保存などの操作が可能です。

## 2. 機能一覧

### 2.1 画面キャプチャ

| キャプチャ方法 | 説明 | 既定のホットキー |
| --- | --- | --- |
| **スクリーン選択** | マルチスクリーン環境でキャプチャする画面をクリックして選択します。シングルスクリーン時は即時キャプチャされます。 | `Ctrl + Shift + Q` |
| **ウィンドウ選択** | マウスカーソル下のウィンドウを自動検出し、クリックで選択してキャプチャします。 | `Ctrl + Shift + W` |
| **範囲選択** | マウスドラッグで自由に矩形領域を指定してキャプチャします。 | `Ctrl + Shift + E` |

キャプチャ中はメインウィンドウが自動的に最小化され、操作の妨げになりません。  
キャプチャ中の操作を中断するには `Esc` キーを押します。

### 2.2 画像プレビューと編集

キャプチャした画像はメインウィンドウのプレビューエリアに表示され、以下の編集操作が可能です。

#### 2.2.1 切り出し（クロップ）

- プレビュー画像上でマウスドラッグするだけで切り出しモードが開始されます
- 選択領域は 8 方向ハンドルでリサイズ可能
- 選択領域内をドラッグして移動可能
- アプリ内のボタンまたはコンテキストメニューから「切り出し」を実行

#### 2.2.2 自動ウィンドウ検出切り出し

- 画像内のウィンドウ領域を背景色から自動判定して切り出します
- 四隅の背景色を基準に、同じ色が連続する領域を除去する方式

#### 2.2.3 クリップボード連携

| 操作 | ショートカット |
| --- | --- |
| 画像をクリップボードにコピー | `Ctrl + C` |
| クリップボードから画像を貼り付け | `Ctrl + V` |

クリップボードに画像がない場合は貼り付けできません。

#### 2.2.4 アンドゥ

- 最大 5 段階のアンドゥに対応
- 切り出し操作前の状態に戻せます

#### 2.2.5 ズーム

- 25% ～ 500% の段階的ズーム（18 段階）
- 自動フィットモード（画像をパネルに合わせる）
- マウスホイール＋`Ctrl` キーで拡大／縮小
- マウスホイール＋`Shift` キーで水平スクロール
- マウスホイールのみで垂直スクロール
- ステータスバーに現在の拡大率を表示

#### 2.2.6 ルーペ（拡大鏡）

- 画像上のマウス位置を拡大表示
- 3 モード切替：常時表示 / 非表示 / 範囲選択中のみ自動表示
- 拡大率 4 ～ 32 倍（偶数推奨）
- サイズ 64 ～ 512 ピクセル
- 十字線と外枠の色・太さをカスタマイズ可能

#### 2.2.7 画像配置

- **中央寄せ**: 画像が小さい場合でもパネル中央に表示
- **左上寄せ**: 画像をパネルの左上に配置（スクロール可能）

### 2.3 ファイル保存

#### 2.3.1 ファイル名テンプレート

テキストボックスにテンプレート文字列を入力して保存ファイル名を決定します。

| プレースホルダ | 置換内容 | 例 |
| --- | --- | --- |
| `{date}` | 日付（yyyyMMdd 形式） | `20250101` |
| `{time}` | 時刻（HHmmss 形式） | `120000` |

**数値の自動インクリメント**:  
テンプレート内の右端の数値連続が自動的に +1 されます。  
例: `screenshot_01.png` → `screenshot_02.png` → ... → `screenshot_99.png` → `screenshot_100.png`

- 桁数は維持されます（`01` → `02`、`099` → `100`）
- 複数の数値がある場合は一番右の数値の塊がインクリメント対象
- テンプレートは次回起動時に自動復元されます

#### 2.3.2 保存形式

拡張子によって自動判定されます（既定は PNG）。

| 拡張子 | 形式 |
| --- | --- |
| `.png` | PNG |
| `.jpg` / `.jpeg` | JPEG |
| `.bmp` | BMP |
| `.gif` | GIF |

#### 2.3.3 保存先フォルダ

- メニューバー「設定」→「保存先のフォルダ」から任意のフォルダを選択可能
- 既定の保存先: `%USERPROFILE%\Pictures\SaveEvidence`
- 設定は自動保存され、次回起動時に復元されます
- ツールバーのリンクラベルをクリックするとエクスプローラーで開く
- リンクラベルにマウスを乗せるとフルパスが ToolTip 表示される

#### 2.3.4 上書き確認

同名ファイルが存在する場合は上書き確認ダイアログが表示されます。

### 2.4 フォルダビューアー

保存先フォルダの内容を表示するモードレスダイアログです。

- **シェルアイコン表示**: ファイル読み込み時に OS のシェルアイコンを高速に仮表示
- **非同期サムネイル読み込み**: 画像ファイルのサムネイルをバックグラウンドで非同期生成し、順次差し替え表示
- **5 種類の表示モード**: 特大 / 大 / 中 / 一覧 / 詳細
- **ソート機能**: ファイル名の昇順・降順切替
- **ダブルクリック**: 既定のアプリケーションでファイルを開く
- **ウィンドウ位置保存**: 次回起動時に位置・サイズを復元

### 2.5 グローバルホットキー設定

メニューバー「設定」→「グローバルホットキー」から各キャプチャタイプのホットキーを変更できます。

- 修飾キー: `Ctrl` / `Alt` / `Shift` / `Win`（複数組合せ可）
- 「設定」ボタンをクリックして実際のキーボード入力で設定
- 設定内容はボタンの右側に表示されます
- 設定は自動保存され、次回起動時に復元されます

### 2.6 表示設定

メニューバー「設定」→「表示設定」から以下の項目を変更できます。

| カテゴリ | 設定項目 |
| --- | --- |
| **画像配置** | 中央寄せ / 左上寄せ |
| **キャプチャ方式** | PrintWindow（DWM 直接取得）/ CopyFromScreen（従来方式） |
| **選択枠色（キャプチャ時）** | キャプチャ選択画面の境界線色 |
| **選択枠色（切り出し時）** | プレビュー切り出しの境界線色 |
| **ルーペ拡大率** | 4 ～ 32 倍 |
| **ルーペサイズ** | 64 ～ 512 ピクセル |
| **ルーペ十字線色** | ルーペの十字線の色 |
| **ルーペ外枠色** | ルーペの外枠の色 |
| **ルーペ外枠太さ** | 外枠の太さ（ピクセル） |
| **フォルダビューアーアイコンサイズ** | 特大 / 大 / 中 |

## 3. システム要件

| 項目 | 要件 |
| --- | --- |
| **OS** | Windows 10（17763+）以降 |
| **.NET ランタイム** | .NET 10.0 以降 |
| **アーキテクチャ** | x64 |

### 3.1 リリースビルド

公開ビルドでは以下の設定が有効になります。

- **単一ファイル実行可能ファイル**（`PublishSingleFile`）
- **ReadyToRun コンパイル**（起動時間短縮）
- **自己完結型配置**（ランタイム同梱）
- **アンマネージドコードのトリミング**（不要コードの削減）

## 4. インストール方法

### 4.1 ソースからのビルド

```powershell
git clone https://github.com/your-org/SaveEvidence.git
cd SaveEvidence
dotnet publish src/app/app.csproj -c Release -o Publish
```

`Publish/SaveEvidence.exe` を実行してください。

### 4.2 単一ファイル実行

リリースビルドの出力は単一の `SaveEvidence.exe` です。任意のフォルダに配置して実行してください。

## 5. 操作方法

### 5.1 基本操作フロー

```text
1. アプリケーション起動
   └── SaveEvidence.exe を実行
   └── 前回終了時のウィンドウ位置・サイズが復元

2. 画面キャプチャ
   └── ツールバーのボタンまたはグローバルホットキーでキャプチャ開始
   └── キャプチャ中はメインウィンドウが最小化
   └── Esc キーでキャンセル

3. 画像の確認と編集
   └── キャプチャした画像がプレビューエリアに表示
   └── マウスホイール Ctrl+ でズーム調整
   └── 必要に応じて切り出しや自動ウィンドウ検出を実行

4. 画像の保存
   └── ファイル名テンプレートを確認・編集
   └── 「保存」ボタンまたは Ctrl+S で保存
   └── 保存後にファイル名の数値が自動インクリメント
```

### 5.2 画面構成

```text
┌──────────────────────────────────────────────────┐
│  タイトルバー                                      │
├──────────────────────────────────────────────────┤
│  メニューバー                                      │
│  [ファイル] [編集] [キャプチャ] [表示] [設定]       │
├──────────────────────────────────────────────────┤
│  ツールバー                                        │
│  [全画面][ウィンドウ][範囲][自動切出][保存]        │
│  ファイル名テンプレート: [______________________]  │
│  保存先: 📁 SaveEvidence                          │
├──────────────────────────────────────────────────┤
│                                                  │
│              プレビューエリア                      │
│                                                  │
│                                                  │
├──────────────────────────────────────────────────┤
│  ステータスバー                                    │
│  拡大率: 100% | 1920,1080 | 未保存 | 中央寄せ     │
└──────────────────────────────────────────────────┘
```

### 5.3 キャプチャ選択画面の操作

| モード | 操作方法 |
| --- | --- |
| **スクリーン選択** | マウス移動でハイライトされるスクリーンをクリックして確定 |
| **ウィンドウ選択** | マウス移動で自動検出されるウィンドウをクリックして確定 |
| **範囲選択** | マウスドラッグで矩形を描画し、ボタンを離して確定 |

### 5.4 キーボードショートカット一覧

| ショートカット | 機能 |
| --- | --- |
| `Ctrl + Shift + Q` | スクリーン選択キャプチャ |
| `Ctrl + Shift + W` | ウィンドウ選択キャプチャ |
| `Ctrl + Shift + E` | 範囲選択キャプチャ |
| `Ctrl + C` | 画像をクリップボードにコピー |
| `Ctrl + V` | クリップボードから画像を貼り付け |
| `Ctrl + S` | 画像を保存 |
| `Ctrl + Z` | アンドゥ（切り出し操作を元に戻す） |
| `Ctrl + マウスホイール` | ズーム（拡大／縮小） |
| `Shift + マウスホイール` | 水平スクロール |
| `Esc` | キャプチャ中：キャンセル |
| `Ctrl + +/-` | ズームイン／アウト |

## 6. 設定ファイル

設定は以下のパスに JSON 形式で保存されます。

```text
%LOCALAPPDATA%\SaveEvidence\settings.json
```

例:

```json
{
  "SelectScreenHotKey": { "Modifiers": 6, "Key": 81 },
  "WindowSelectHotKey": { "Modifiers": 6, "Key": 87 },
  "AreaSelectHotKey": { "Modifiers": 6, "Key": 69 },
  "FileNameTemplate": "screenshot_{date}_{time}.png",
  "SaveFolderPath": "C:\\Users\\...\\Pictures\\SaveEvidence",
  "CenterAlign": true,
  "CaptureMode": 0,
  "LoupeZoomLevel": 8,
  "LoupeSize": 170
}
```

保存される設定項目:

- 各キャプチャタイプのグローバルホットキー
- ファイル名テンプレート文字列
- 保存先フォルダパス
- ウィンドウ位置・サイズ（メイン / フォルダビューアー）
- 表示設定（色、ルーペ、配置、アイコンサイズ）
- キャプチャ方式（PrintWindow / CopyFromScreen）
- フォルダビューアーの表示モード・ソート順・アイコンサイズ

## 7. アーキテクチャ概要

SaveEvidence は、Windows Forms 上で CommunityToolkit.Mvvm を使用した MVVM ライクなアーキテクチャを採用しています。

```text
┌──────────────────────────────────────────┐
│              View（フォーム）              │
│  MainForm / SelectionForm / SettingsForm  │
│  HotkeyForm / FolderViewForm             │
└────────────────┬─────────────────────────┘
                 │ DataBindings & Events
┌────────────────▼─────────────────────────┐
│          ViewModel                        │
│  MainViewModel / FolderViewViewModel     │
└────────────────┬─────────────────────────┘
                 │
┌────────────────▼─────────────────────────┐
│          Model ＆ Service                 │
│  CaptureManager / HotKeyManager           │
│  Settings / SettingsService               │
│  ImageProcessor / FileNameTemplate        │
│  CropSelection                            │
└──────────────────────────────────────────┘
```

## 8. トラブルシューティング

| 問題 | 原因と対処 |
| --- | --- |
| **ホットキーが反応しない** | 他のアプリケーションが同じホットキーを使用している可能性があります。設定画面から別のキー組合せに変更してください。 |
| **キャプチャ画像が真っ黒** | PrintWindow 方式でモダンアプリ（WinUI）をキャプチャする際に発生する場合があります。表示設定から CopyFromScreen 方式に切り替えてください。 |
| **ウィンドウ選択で正しく取得できない** | ハードウェアアクセラレーションが有効なコンテンツ（動画・3D）は CopyFromScreen 方式を推奨します。 |
| **設定ファイルが壊れた** | `%LOCALAPPDATA%\SaveEvidence\settings.json` を削除してアプリを再起動すると、デフォルト設定で再作成されます。 |
