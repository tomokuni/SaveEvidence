画面キャプチャして連番で保存するアプリを作成します。

キャプチャの種類は、スクリーン選択、アクティブウィンドウ、範囲選択の3種類をサポートします。
キャプチャした画像をアプリに表示して確認できます。
アプリに表示された画像に対して以下の操作が可能です。
- クリップボードへのコピー
- クリップボードから画像の貼り付け
- 範囲選択を切り出してアプリに表示
- 画像内のウィンドウ領域を自動判定して切り出してアプリに表示
- 保存

キャプチャは以下の方法で行えます。
- グローバルホットキーを押すことでキャプチャできます。
- アプリ内のボタンを押すことでキャプチャできます。

グローバルホットキーは、設定で変更可能ですが、規定では以下のように設定されています。
- スクリーン選択: `Ctrl + Shift + Q`
- アクティブウィンドウ: `Ctrl + Shift + W`
- 範囲選択: `Ctrl + Shift + E`
グローバルホットキーの変更は、メニューバーの「設定」→「グローバルホットキー」で行えます。
- ダイアログが表示され、各キャプチャのグローバルホットキーを変更できます。
- グローバルホットキー ダイアログの画面構成
  - スクリーン選択 キャプチャのホットキー設定
    - 設定ボタンを押して、ホットキーを入力すると、設定できます。
    - 設定されているホットキーは、ボタンの右に表示されます。
  - アクティブウィンドウ キャプチャのホットキー設定
    - 設定ボタンを押して、ホットキーを入力すると、設定できます。
    - 設定されているホットキーは、ボタンの右に表示されます。
  - 範囲選択 キャプチャのホットキー設定
    - 設定ボタンを押して、ホットキーを入力すると、設定できます。
    - 設定されているホットキーは、ボタンの右に表示されます。

キャプチャに使用するアプリ内のボタンは以下になります。
- スクリーン選択
- アクティブウィンドウ
- 範囲選択
以下のボタンもあります。
- 画像内のウィンドウ領域を自動判定して切り出し

キャプチャ動作中の動作・操作
- キャプチャ動作中はアプリのウィンドウを最小化します。
- キャプチャ動作中に Esc キーを押すと、キャプチャを中止できます。

アプリに表示された画像は、ファイル名テンプレートテキストを基にファイル名を決定して保存します。
ファイル名テンプレートテキストは、以下のように設定できます。
- フォームの上部にあるテキストボックスに入力することで設定できます。
- 日付と時刻を含める: `screenshot_{date}_{time}.png`
- 数値はプレースフォルダと現在の数値を表します。
  - インクリメントした後の数値が次のファイル名テンプレートテキストになります。
    - 現在表示されているのが `screenshot_1.png` の場合、保存するのは `screenshot_1.png` になります。
    - 保存後にインクリメントされ、`screenshot_2.png` になります。
  - 数値の塊が実質的なプレースフォルダになります。
  - 1 桁の数値の塊は、1 桁の数値を表します。ただし、9 の次は 10 の 2 桁になります。
  - 2 桁の数値の塊は、2 桁の数値を表します。01 の次は 02、 02 の次は 03、09 の次は 10、99 の次は 100 になります。
  - 複数の数値の塊がある場合は、一番右の数値の塊を対象にインクリメントします。
- 最後のテンプレートテキストは、設定ファイルに自動保存され、次回起動時に読み込まれます。
保存先のフォルダは、以下のように設定できます。
- 保存先のフォルダは、メニューバーの「設定」→「保存先のフォルダ」で設定できます。
  - フォルダ選択ダイアログが表示され、保存先のフォルダを選択できます。
- 保存先のフォルダは、設定ファイルに自動保存され、次回起動時に読み込まれます。
- 保存先設定ボタンの ToolTip に現在の保存先フォルダのパスが表示されます。
- 保存先設定ボタンの右に、リンクラベルがあり、現在の保存先フォルダ名が表示されます。
- リンクラベルをクリックすると、保存先フォルダをエクスプローラーで開きます。

画面構成
- タイトルバー
- メニューバー
  - 設定
    - 保存先のフォルダ
    - グローバルホットキー
- ツールバー
  - フルスクリーン キャプチャボタン
  - ウィンドウ選択 キャプチャボタン
  - 選択範囲 キャプチャボタン
  - 画像内のウィンドウ領域を自動判定して切り出しボタン
  - ファイル名テンプレートテキスト
  - 保存先のフォルダリンク
- プレビューエリア

公開時は、以下の設定を有効にするので、問題ないようにコードを作成します。
- 単一ファイルの作成
- ReadyToRun コンパイルを有効にする
- 未使用のコードのトリミング

MVVMのように、プロパティを変更すると、UIが自動的に更新されるように設計します。
フォルダやファイルの構成も MVVM に従って、View、ViewModel、Model のように分けて作成します。
WinForms で MVVM を実現するのが難しい場合には、ある程度妥協します。

アプリで例外発生時は、１箇所でキャッチして、例外内容をコンソールに出力するようにします。
また、キャプチャ動作中に例外発生時は、画面の暗転等を基に戻して、Windows等の操作ができるようにします。
